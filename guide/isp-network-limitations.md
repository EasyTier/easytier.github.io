# 运营商网络限制

以下内容主要基于部分地区用户的实际观察与反馈，用于帮助理解为什么 EasyTier 在某些网络环境下难以建立稳定的直连。
不同地区、运营商、套餐和时间段的情况可能差异很大，请不要将其视为对所有网络环境的一般性结论。

在跨省、跨运营商，或高峰时段的场景中，部分用户可能会遇到以下现象：

1. 上传带宽被限制，例如上行被压到较低水平，这会明显影响 MC 用户、串流用户和 NAS 用户。
2. 丢包率升高，严重时会直接影响游戏体验、远程控制和大文件传输。
3. 延迟增加，同一条路径在高峰时段可能比平时高出很多。

::: danger 警告
这些现象会明显影响 EasyTier 建立和维持 P2P 连接的质量，严重时会导致只能走中继链路。
:::

### 其他常见限制

1. 分时段限制：某些地区在晚高峰更容易出现限速、丢包或高延迟，也可能只出现其中一部分。
2. 限制连接数：当设备较多或连接较多时，可能无法建立新连接，表现为断流或连接失败。
3. 限制打洞行为：即使 NAT 条件看起来较好，运营商侧策略也可能导致 P2P 无法建立。
4. 限制流量：例如对每日或每月上传流量设置阈值，超出后进一步限速。
5. 按协议差异化处理：在一些网络环境里，UDP 流量会比 TCP 更容易受到限制，这会直接影响打洞成功率与链路质量。

### 关于跨国流量

跨国流量通常比国内链路更复杂，更容易受到拥塞、绕路、策略限制或质量波动的影响。

::: warning 注意
不同地区、不同运营商，甚至同一运营商的不同套餐，实际表现都可能不同。有些地区几乎不受影响，有些地区则限制较为明显。
:::

::: danger 警告
在部分地区，即使是非高峰时段、同城、同运营商的链路，只要上传流量持续较高，依然可能触发限速或连接质量下降。
:::

## 那么影响了什么呢？

### 游戏用户

拿我的世界（MC）举例，MC 的整合包对带宽要求较高，本来高延迟和丢包体验已经很差，限速的话甚至会无法联机，比如经典的
连接中断错误。

对带宽没有要求的游戏体验也会很差。

### NAS 用户

虽然 NAS 用户通常对延迟不敏感，但限速会显著拉长传输时间；丢包则会带来更多重传，严重时可能导致连接中断或应用层校验失败。

### 串流用户

一般串流用户对画质和帧率有着较高的要求，这就需要较大的带宽，往往会较为严重的造成卡顿、掉线等问题。

## 什么是跨省、跨网？

跨省指的是流量在不同的省份之间进行传输，例如河南省——河北省。

跨网指的是流量在不同的运营商之间进行传输，例如中国电信——中国联通。

## 为什么协议会影响 P2P 的连接质量？

EasyTier 同时支持 UDP 和 TCP 打洞，但在很多网络环境中，UDP 更依赖运营商对无连接流量和 NAT 行为的放行，
因此通常更容易受到限制。如果运营商对 UDP 或特定打洞行为做了额外限制，P2P 成功率和连接质量都会下降。

## 有什么解决方法吗？

1. 优先尝试更稳定的链路，例如允许 TCP 打洞，或在必要时直接放弃 P2P，改走中继。

2. 通过服务器进行中转。在 EasyTier 中，可根据需要启用 [`--disable-p2p`](/guide/network/configurations#完整配置选项)
   等配置，只通过已知节点转发数据包。

::: warning 注意
公共服务器的带宽和稳定性通常有限。如果需要大量传输数据，建议自建或租用带宽更充足的中继节点，尤其是 NAS、串流和大型游戏
整合包场景。
:::

3. 咨询运营商是否有更适合高上行流量或跨网传输的套餐。

4. 如果你认为链路质量异常且长期影响使用，可以保留测速和抓包证据后向运营商客服或监管渠道反馈。

5. 跨国用户通常需要更稳定的中继节点，必要时可结合专线、就近中继或其他跨境网络方案。

## 为什么网络环境会变成这样？

成因通常与网络成本、流量结构变化、运营商治理策略以及不同业务的带宽模型有关，具体情况会因地区和套餐而异。
对于终端用户而言，更重要的是识别问题是否存在，并选择合适的组网策略。
